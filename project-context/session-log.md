## Session Guidelines

### Starting a Session

1. Update "Active Session" section with start time and goal
2. Review `status.md` and `tasks.md` for context
3. Note your planned approach

### During a Session

- Update progress notes as you work
- Document significant decisions in `decisions.md`
- Note any blockers in `status.md`
- Update `tasks.md` as you complete items

### Ending a Session

1. Move active session to "Session History"
2. Fill in accomplishments and handoff notes
3. Update `status.md` with current state
4. Update `tasks.md` with completed items
5. Clear "Active Session" section

### What Makes a Good Session Log

- **Clear goal statement**: What you set out to do
- **Specific accomplishments**: What you actually did
- **Decision context**: Why you made key choices
- **Handoff clarity**: What someone needs to know to continue
- **Honest status**: Issues encountered and unresolved items

## Session Templates

### For Feature Development

```markdown
**Goal**: Implement [feature name]
**Accomplishments**:

- Created [component/module]
- Implemented [functionality]
- Added tests for [scenarios]
  **Decisions**: [Reference decisions.md entries]
  **Handoff**: [What remains to be done]
```

### For Bug Fixing

```markdown
**Goal**: Fix [bug description]
**Accomplishments**:

- Identified root cause: [explanation]
- Implemented fix in [file/module]
- Added regression test
  **Decisions**: [Approach taken and why]
  **Handoff**: [Follow-up items if any]
```

### For Investigation

```markdown
**Goal**: Investigate [issue/question]
**Accomplishments**:

- Analyzed [component/code]
- Findings: [what you learned]
- Recommendations: [proposed approach]
  **Handoff**: [Next steps based on findings]
```
